## 2) Component Inventory And Purpose

- `BUTDDMList`
  - Base dropdown list with animated header button, selectable list, and decorative 20dp bottom spacer.
- `BUTDDMListAction`
  - Same as above, but bottom area is a functional single `Clear All` button (40dp).
- `BUTDDMListActionDual`
  - Same as above, with two 40dp bottom buttons: `Clear All` + `Select All`.

All three inherit almost all behavior from `BUTDDMListBase`.

---

## 3) Shared Behavior Contract (BUTDDMListBase)

## Selection model

- Modes:
  - `SINGLE`
  - `MULTI`
- Internal state:
  - `items: List<String>`
  - `selectedItems: MutableSet<Int>`
  - `disabledPositions: Set<Int>`
- Public API (important for web parity):
  - `setItems(items)`
  - `setFilterName(name)`
  - `setSelectionMode(mode)`
  - `setSelectedPositions(positions)`
  - `getSelectedPositions()`
  - `getSelectedItems()`
  - `clearSelection()`
  - `selectAll()`
  - `setDisabledPositions(positions)`
  - `setOnSelectionChangedListener(listener)`
  - `closeDropdown()`
  - `setColorFill(fill)`
  - `refreshTheme()`

## Header button text rules

- In `SINGLE` mode:
  - no selection: `filterName` if provided, else `Select option`
  - selected: selected item text
- In `MULTI` mode:
  - no selection: `0 selected`
  - all selected + `filterName` present: `All <filterName> Selected`
  - partial: `<count> selected`

## Open/close and animation

- Header toggles dropdown open/closed.
- Duration: `300ms`.
- Two animations run in parallel:
  - header corner radius transition:
    - closed: pill
    - open: top-rounded
  - list + bottom area height/alpha transition
- Chevron icon:
  - closed: right chevron
  - open: down chevron
- Haptic feedback on open and close.

## Height rules

- Row height fixed at `40dp`.
- Max visible rows before scrolling: `8`.
- Expanded list height:
  - if items > 8: `8 * 40dp` (scrollable)
  - else: `items * 40dp`

## List interactions

- Row click toggles selection (unless row is disabled).
- `SINGLE`: selecting one clears previous.
- `MULTI`: independent toggles.
- Adapter notifies only changed rows where possible.
- Selection callback is posted asynchronously (`post { ... }`) to keep taps responsive.

## 4) Variant-Specific Behavior

## A) `BUTDDMList` (decorative bottom)

- Bottom area is a visual spacer (`20dp`).
- Spacer is non-interactive.
- Styled as pressed-like bottom-rounded strip to complete dropdown silhouette.

## B) `BUTDDMListAction` (single action)

- Bottom area: one 40dp bottom-rounded button labeled `Clear All`.
- Visible only when:
  - dropdown is open
  - `selectionMode == MULTI`
- Action:
  - clears selection.

## C) `BUTDDMListActionDual` (dual action)

- Bottom area: horizontal container with two 40dp buttons:
  - left: `Clear All`
  - right: `Select All`
- Visible only when:
  - dropdown is open
  - `selectionMode == MULTI`
- Both buttons are bottom-rounded on outer edges and square on inner edges.
- Uses slight negative inner margins to visually merge seam and keep shadows consistent.

## 5) Visual Specification

## Dimensions

- Header button: `40dp` height
- Row item: `40dp` height
- Bottom spacer (basic variant): `20dp`
- Bottom action area (action variants): `40dp`
- Row max viewport: `320dp` (`8 * 40dp`)

## Typography

- Header text:
  - `18sp`, Geologica variable, `wght 500`
- Row text:
  - `14sp`, Geologica variable, `wght 400`
- Horizontal content padding in header/rows:
  - `18dp` left/right

## Shape system

- Closed header: full pill radii.
- Open header: top corners rounded, bottom corners square.
- Rows: square edges (`BUTSquaredListWide`).
- Bottom action/spacer gives rounded bottom silhouette.

## Shadows/states

- Header closed uses outer + inner shadow treatment.
- Header open uses flatter pressed-style background (outer shadow suppressed on header itself).
- Container shadow is applied to full open dropdown shell at animation end.
- Row selected state uses pressed-look drawable (not a separate checkmark-only state).
- Selected rows also show checkmark icon.

## 6) Mapping To Existing Web BUT Primitives

Use your existing web `BUT*` styles as the foundation:

- Header shell:
  - closed -> equivalent of wide/pill `BUTWideButton`
  - open -> equivalent top-rounded pressed state (`BUTTopRoundedWideButton` style behavior)
- List rows:
  - `BUTSquaredListWide` equivalent (square 40px rows)
- Bottom area:
  - spacer variant -> pressed bottom rounded strip
  - action variant -> one `BUTBottomRoundedWideButton`
  - dual variant -> two adjoining bottom-rounded buttons with inner square seam

Token reuse:

- Fill family via same style families (`DEFAULT`, `SECONDARY`, `RED_ACTION`, `GREEN_ACTION`, `ACCENT`, `FAVORITE`)
- Use same text/shadow tokens from your web button theme layer.

## 7) Recommended Web Component Architecture

Implement one shared component and three wrappers:

- `DdmBase`
- `DdmList` (decorative bottom)
- `DdmListAction` (clear button)
- `DdmListActionDual` (clear + select all)

## Suggested internal parts

- `DdmHeaderButton`
  - text label + chevron + open/closed animation state
- `DdmListViewport`
  - virtualized or standard scroll list capped at 8 rows
- `DdmRowButton`
  - square row primitive with selected style + checkmark
- `DdmBottomArea`
  - strategy by variant (`spacer`, `singleAction`, `dualAction`)

## 8) CSS Blueprint (Web)

```css
.ddm {
  position: relative;
}

.ddm__header {
  height: 40px;
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 0 18px;
}

.ddm--closed .ddm__header {
  border-radius: 999px;
}

.ddm--open .ddm__header {
  border-radius: 45px 45px 0 0;
}

.ddm__panel {
  overflow: hidden;
}

.ddm__listViewport {
  max-height: 320px; /* 8 rows */
  overflow-y: auto;
}

.ddm__row {
  height: 40px;
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 0 18px;
  border-radius: 0;
}

.ddm__bottomAction,
.ddm__bottomDual button {
  height: 40px;
}
```

Animation targets to match Android:

- `header border-radius` (closed <-> open)
- `panel height`
- `panel opacity`
- duration `300ms`, ease-in-out equivalent.

## 9) JavaScript Behavior Rules (Web Parity)

- Toggle open/close from header click.
- Maintain `selectionMode` (`single` vs `multi`) exactly.
- Apply same header-label rules.
- Keep disabled rows non-interactive.
- Fire `onSelectionChanged(selectedPositions, selectedItems)` after internal state update.
- `setSelectionMode('single')` should collapse multi selection down to first selected.
- `setItems` resets selection by default (as Android does).
- `closeDropdown()` should close only if currently open.
- Action buttons:
  - clear all -> deselect all
  - select all -> select all rows
- Action areas visible only when open + multi.

## 10) Implementation Matrix

- `BUTDDMList` -> `DdmList`:
  - `bottomType = spacer20`
- `BUTDDMListAction` -> `DdmListAction`:
  - `bottomType = clearAll40`
- `BUTDDMListActionDual` -> `DdmListActionDual`:
  - `bottomType = clearAllSelectAll40`

All should share the same state engine and row rendering.

---

## 11) QA Checklist (Web)

- Header closed/open shape matches Android.
- Open/close animation duration is ~300ms.
- Chevron direction updates correctly.
- Max visible rows is 8 before scrolling.
- `single` mode allows only one selected row.
- `multi` mode supports multi toggles.
- Disabled rows cannot be selected and look visually disabled.
- Header label text matches Android rules in all states.
- Action area visibility rules match variant + mode + open state.
- `clear all` and `select all` work and emit callbacks.
- Theme/fill family switching updates header, rows, icons, and actions.
